1. Architecture Overview

EduTrack follows a client-server architecture where React handles the frontend UI and AppWrite acts as the complete backend-as-a-service (BaaS) layer. A fingerprint scanner device at each classroom kiosk interfaces with the browser via a hardware SDK or the WebAuthn API.

4. Fingerprint Integration

4.1 Approach

Fingerprint scanning requires a USB biometric scanner device at each classroom kiosk. The two recommended approaches are:

Approach

How It Works

When to Use

WebAuthn (FIDO2) API

Browser-native API that delegates biometric matching to the OS. Works with Touch ID, Windows Hello, or a FIDO2-certified USB scanner.

Modern hardware, no vendor lock-in, best privacy (templates never leave device)

Vendor SDK (e.g. DigitalPersona, SecuGen)

Hardware vendor provides a JavaScript or REST bridge SDK. App communicates with a local agent or browser extension that reads the scanner.

Legacy or budget hardware that does not support FIDO2

4.3 Fingerprint Data Flow

  • Enrollment (one-time, done by Admin): Admin selects student → kiosk prompts finger scan → WebAuthn creates a credential → public key stored in AppWrite Storage linked to student profile. Raw biometric never leaves the device.
  • Check-In (every class): Student taps scanner → WebAuthn authenticates against stored credential → on success, app calls AppWrite to write an attendance record for the active session.
  • Failure Fallback: If scan fails 3 times, teacher is prompted to manually override.
