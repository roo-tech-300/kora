EduTrack

Fingerprint Attendance Management System

Product Requirements Document (PRD)

Version 1.0 | April 2026

1. Product Overview

EduTrack is a school attendance management system that leverages biometric fingerprint scanning to record student presence in classes. The system replaces manual roll calls with fast, accurate, tamper-proof attendance logging — giving administrators, teachers, and parents real-time insight into student attendance.

2. Goals & Objectives

2.1 Business Goals

  • Eliminate manual attendance-taking errors and inefficiencies
  • Provide real-time, accurate attendance data to school staff
  • Generate automated attendance reports for parents and administration
  • Reduce time spent on administrative tasks at the start of each class

2.2 Product Goals

  • Allow students to check in to class by scanning their fingerprint in under 3 seconds
  • Give teachers a live view of who is present or absent during a class
  • Enable admins to manage students, classes, teachers, and generate reports
  • Send automated absence alerts to guardians
  • Maintain a secure, auditable log of all attendance events

5. User Stories

5.1 Admin

  • As an admin, I want to register a new student and enroll their fingerprint so that they can check into classes.
  • As an admin, I want to create classes and assign teachers to them so that attendance sessions can be opened.
  • As an admin, I want to view attendance reports for all classes so that I can monitor school-wide trends.
  • As an admin, I want to set an absence threshold so that I am automatically alerted when a student is at risk.

5.2 Teacher

  • As a teacher, I want to open a class session so that my students can begin checking in via fingerprint.
  • As a teacher, I want to see a live dashboard during class showing who has and hasn't arrived yet.
  • As a teacher, I want to manually mark a student present if the fingerprint scanner fails.
  • As a teacher, I want to export an attendance report for my class to share with administration.

5.3 Student

  • As a student, I want to place my finger on the scanner at the classroom kiosk to mark myself as present.
  • As a student, I want to receive confirmation on screen that my attendance was recorded successfully.

6. Non-Functional Requirements

6.1 Performance

  • Fingerprint matching and attendance recording must complete within 3 seconds of scan.
  • The dashboard must reflect check-ins within 2 seconds (near real-time).

6.2 Security

  • Fingerprint templates are stored as encrypted hashes — raw biometric data is never stored.
  • All user authentication is handled through AppWrite Auth with role-based access control.
  • All API communication uses HTTPS/TLS.

6.3 Reliability

  • The system must support offline queuing: if connectivity is lost, check-ins are stored locally and synced when reconnected.
  • Target uptime: 99.5% during school hours.

6.4 Usability

  • The kiosk interface must be operable by any student in under 5 seconds with no training.
  • All admin and teacher interfaces must be accessible on desktop and tablet.

6.5 Scalability

  • The system must support up to 2,000 students and 100 concurrent class sessions per deployment.

7. Constraints & Assumptions

  • Each classroom has a fingerprint scanner device connected to a dedicated kiosk computer or tablet.
  • Fingerprint hardware provides a JavaScript SDK or browser-compatible interface.
  • School has reliable internet access (with offline fallback for kiosk).
  • AppWrite is self-hosted or cloud-hosted by the school's IT department.
  • Initial release targets a single school deployment; multi-school support is a future phase.

8. Out of Scope (V1)

  • Mobile app for students or parents
  • Face recognition or other biometric methods
  • Grade management or LMS integration
  • Multi-school or district-level admin
  • SMS notifications (email only in V1)
